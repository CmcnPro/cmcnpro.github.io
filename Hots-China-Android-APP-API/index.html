风暴英雄API分析
2017-07-20

随身风暴英雄API分析,网易出品APP,非暴雪官方出品

申明

本文章仅提供API分析,可能随着官方APP更新而失效.请勿将API滥用,本文章只提供交流学习,一切后果,由API调用者负责,与本文作者无关.
本API仅对国服有效 ,本文会慢慢更新.

序言

在风暴2.0时,大比利说过会开放风暴英雄的API,但是到目前(2017年7月20日),并没有看到任何官方API出来.在2014年,暴雪在开启风暴英雄测试的时候,暴雪中国的运营商网易公司,当时就推出了一款手机APP随身风暴英雄,当时可以在上面抽激活码等等,然后因为不是暴雪出品,网易也确实没有放太多的精力在该APP上面,但是在2.0之后,网易多次更新了该APP,并且带来了一些新的功能,如战绩查询等等.
虽然网易在更新该APP,但是目前一些API已经实现的功能,却没有在APP上实现,外加该APP在Android上权限有一定的需求,所以我选择自己撸羊毛,自己来搞.

战绩查询

需要用战网账户对随身风暴英雄授权获取UID
GET http://account.bnet.163.com/battlenet/login?inner_client_id=hosapp&inner_redirect_uri=http://sh.hosapp.blz.netease.com/hosapp-1.0/nebnlogin

参数 解释
uid 必须,查询的用户
hero 非必需,查询的英雄
mapId 非必需,查询的地图
gameMode 非必需,查询的游戏模式
pageSize 必须,为返回的比赛的数量
pageNum 必须

tips
大家通过链接可以看出,因为不是暴雪出品,所以实际访问的是网易的服务器,虽然暴雪中国也是用的网易的服务器233,个人怀疑其实是网易自己将比赛记录导出来,所以并不能查看所以比赛记录,在APP里写的也只能查看2.0之后的比赛.根据我自己测试,实际只能返回最近的200场比赛,所以直接pageSize=200,pageNum=1.返回所有结果就行了.
hero,mapId,gameMode,为空时,返回所有结果,这几个值可以通过返回的JSON自己查看.

tips
每个值都是必须.后面三个值由上面获取比赛记录返回的JSON获取.
