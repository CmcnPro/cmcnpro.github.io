废话

最近在参加面试,有被拒绝也有拿到offer.面试的还是自学的.NET
成都地区面试的.NET基本都是ASP,然而我根本不会ASP啊啊啊,windows form基本没有,更别说UWP了,我一个都没有找到,目前想法就是转方向吧,UWP什么的自己私下玩下吧.

面试了其实发现了自己最大的问题就是底层知识的欠缺,当初自学,为了快速上手,所以直接是做的项目的方式,边学边做,所以一些底层的东西就直接被忽略了,而人家面试肯定要问底层的啊.

准备面试的时候呢,遇到这样的一个题目,分享出来给大家吧

题目

[不限语言]假设有个班级,有个一位班长和几位小组长以及组员.现在请您设计一个方法或者函数.输入一个学生,得出TA和TA下属的所有学生(如果有)的成绩的总和.

代号 职位 从属 成绩
A 班长 - 90
B 组长 A 88
C 组长 A 84
D 组长 A 89
E 组员 B 45
F 组员 B 77
G 组员 B 33
H 组员 B 98
I 组员 C 99
J 组员 C 67
K 组员 C 80
L 组员 C 75
M 组员 D 87
N 组员 D 79
O 组员 D 78

大家有啥方法不,如果100分算满分,博主给自己的方法只能打60分,因为效率太差了.
