搜索引擎开发日记

2016年8月16日

Demo为一个简单的扒某学院官网的一个爬虫
CLI程序
分为两个CS
主程序Program.cs 和 爬虫核心Core.cs

爬虫核心

使用HttpClent(.NET4.5.1 以上环境)

属性

属性 作用 备注
url 爬虫要爬的网址 -
rawResult 爬虫获取的网页内容 这个是爬虫爬取后的原始数据,未做解析和过滤
text 做过解析过滤后会存入txt中的字符串 字符串应该是url \t url显示的名称\n

方法

方法名 作用 备注
SetUrl(string url) 设置要爬虫的网站 -
GetWebResult 获取原始网页原始数据 仅Demo中使用,实际项目中应隐藏该方法
ResultParser(string resultString) 提取超链接 -
ExportText 导出到TXT文本中 -

开发笔记

在扒取官网的中发现,有点些特殊情况需要处理

  1. 有的链接为相对地址,有的链接是绝对地址,需要将相对地址转化为绝对地址

  2. 学院网站除官网是UTF-8编码,其余全是GBK,且服务器未返回ChatSet,爬虫时应该做好判断

  3. 爬虫的时候很多链接会多次爬到,所以应该对重复的数据不写入数据库中|解决方法:未知

2016年8月17日

感觉之前的架构有问题,准备重构

目前想法:

  1. 将爬虫,分析,IO读写分开,分成独立的部分,原因如下:
    • 之前的爬虫爬一次后就不会再次爬,而无论Google还是bing的蜘蛛都不止一次对同一网站的内容访问获取
    • 之前的方案是爬虫->解析->IO读写.而作为服务,考虑效率等问题,三者应该分开独立,例如爬虫这边在循环的获取,而解析后应该要与之前的获取的进行比较,再进行IO
    • IO读写后需要告知他们有新的数据进来了,分词与倒排索引应该开始
  2. 需要设置一个URL Frontier和重复消除模块
  3. 虽然是为学院官网写搜索引擎,但是我们还是应该要实现多线程来提升爬虫性能
  4. 因为所有引擎,需要考虑网站的权重等问题(虽然详细的解决方案我还不知道怎么弄,但还是写上吧)

2016年8月18日

重构思路

  1. 需要两个本地的数据库,分别存放URL集与URL和文档集合
  2. 设置一个URL Frontier,里面存放当前待抓取的URL
  3. 关于去除重复的URL,因为获取一次页面后,会获得很多URL,可能其中的部分URL在之前已经获取过了,这时通过一个布隆过滤取将重复获取过后的URL去掉,然后将未获取的URL放入URL Frontier中,让爬虫再次获取网页.
  4. 解析页面解析时,需要将相对地址转化为绝对地址
  5. 这时的爬虫是作为服务进行的,所以获取网页时,应该设置好响应超时时间或者其他设置、限制,以保证爬虫能够正常运行

代码笔记

类名 作用 备注
GetWebResult 获取原始网页原始数据的类 下载网页、将下载过的网页存入URL集中
HttpClientHandler 辅助类 让爬虫支持GBK,不乱码
Parser 网页解析类 解析网页,处理数据
BloomFilter 布隆过滤器 用来过滤重复的URL
